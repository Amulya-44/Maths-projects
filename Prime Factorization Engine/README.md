\# ⚙️ Prime Factorization Engine



An optimized C implementation of a mathematical engine that breaks down any integer greater than 1 into its unique prime components (e.g., $60 = 2 \\times 2 \\times 3 \\times 5$). This project highlights fundamental algorithmic optimization, array-free math logic, and the core concept behind modern digital cryptography.



\---



\## 🧠 The Mathematics: Fundamental Theorem of Arithmetic



The \*\*Fundamental Theorem of Arithmetic\*\* states that every integer greater than 1 is either a prime number itself or can be represented as a unique product of prime numbers. 



\### How the Engine Works:

1\. \*\*Strip Evens First:\*\* The program checks if the number is divisible by 2. It continuously divides by 2 until the number becomes odd. This allows the program to skip checking all other even numbers entirely.

2\. \*\*Scan Odd Candidates:\*\* Starting from 3, the program steps up by 2 (`i += 2`), testing odd numbers (3, 5, 7, 9...) up to $\\sqrt{n}$.

3\. \*\*Automatic Prime Filtering:\*\* You don't need a separate primality test! Because we divide out factors as soon as we find them, by the time the loop reaches a composite number like 9, all factors of 3 have already been entirely stripped away. Thus, the condition `num % 9 == 0` will always be false.

4\. \*\*The Remainder Rule:\*\* If the remaining number after the loops finish is greater than 1, that remaining value is guaranteed to be a prime factor.



\---



\## ⚡ Performance \& Complexity Analysis



\### Time Complexity: $\\mathcal{O}(\\sqrt{n})$

The worst-case scenario occurs when the input number is a prime number. However, instead of checking all the way up to $n$, the loop terminates early at $\\sqrt{n}$. If a number has a composite factor, at least one of its factoring pairs must be less than or equal to its square root.



\### Space Complexity: $\\mathcal{O}(1)$

Because the logic processes and prints the prime factors on the fly as they are discovered, the program requires zero arrays or vector structures. It maintains a constant memory footprint regardless of how massive the input integer is.



\---



\## 🌐 Cryptographic Significance



In asymmetric cryptography systems like \*\*RSA\*\*, public keys are generated by multiplying two incredibly massive prime numbers together. While finding the product is instantaneous, reversing the process—taking that massive number and extracting its prime factors—is an incredibly hard computational problem. 



This engine is a micro-demonstration of the exact "brute force" problem that hackers face when trying to break encryption keys!
